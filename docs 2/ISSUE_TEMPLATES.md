# Issue Templates (Agent-Ready)

Use these templates as GitHub issues or local tasks in `docs/issues/`.

## UI Screen
Title: `[USER|ADMIN] Screen <name> — APK parity`
- Objective
- Spec refs (pdf section + screenshot path)
- Acceptance criteria
- Evidence (screenshots + logs)
- Rollback

## Backend Endpoint
Title: `[BACKEND] <METHOD> /api/v1/<route> — contract locked`
- Request/Response
- Error codes
- Tests required
- Evidence (curl + test output)

## DB Migration
Title: `[DB] <change>`
- Migration + seed update
- Verification queries
- Evidence (migrate/seed logs)

## Unknown / Ambiguity
Title: `[UNKNOWN] <topic> — lockdown`
- Unknown
- Where looked
- Safest default
- Config knob proposal
- Decision log requirement
